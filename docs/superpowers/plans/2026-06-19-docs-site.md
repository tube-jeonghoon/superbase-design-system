# Plan 3 — `apps/docs` 문서 사이트 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `@superbase/tokens`와 `@superbase/react`를 소비하는 Next.js 기반의 TDS풍 커스텀 문서 사이트(`@superbase/docs`)를 만든다 — Getting Started, Foundations(토큰 시각화), Components(라이브 렌더), 그리고 light/dark 테마 토글.

**Architecture:** Next.js 15 App Router. 루트 레이아웃이 `@superbase/tokens/css`(CSS 변수)와 `@superbase/react/styles.css`(컴포넌트 스타일)를 전역 로드한다. 사이드바 네비 + 헤더(테마 토글) 셸 안에 페이지를 렌더한다. 테마 토글은 클라이언트 컴포넌트로 `<html data-theme>`를 바꾸고 localStorage에 저장한다. 실제 로직(테마 토글, 토큰 데이터)은 Vitest+RTL로 TDD하고, 페이지 전체는 `next build` 성공으로 검증한다.

**Tech Stack:** Next.js 15, React 19, TypeScript 5, Vitest 2 + jsdom + @testing-library/react. 워크스페이스 의존성 `@superbase/react`, `@superbase/tokens`.

> 전제: Plan 1·2가 `main`에 머지되어 `@superbase/tokens`(exports `./css`)와 `@superbase/react`(exports `.`, `./styles.css`)가 존재한다. 모노레포 루트는 `/Users/jeonjeonghoon/Documents/Personal/Projects/design-library`. `pnpm-workspace.yaml`은 이미 `apps/*`를 포함한다. 루트 `.gitignore`는 `.next/`를 포함한다. Node 22, pnpm 10.27.0.

import { render, screen } from "@testing-library/react";
import userEvent from "@testing-library/user-event";
import { ThemeToggle } from "./ThemeToggle";

describe("ThemeToggle", () => {
  beforeEach(() => {
    localStorage.clear();
    document.documentElement.removeAttribute("data-theme");
  });

  it("renders a switch defaulting to light (unchecked)", () => {
    render(<ThemeToggle />);
    expect(screen.getByRole("switch")).not.toBeChecked();
  });

  it("toggles document data-theme and persists to localStorage on click", async () => {
    render(<ThemeToggle />);
    await userEvent.click(screen.getByRole("switch"));
    expect(document.documentElement.getAttribute("data-theme")).toBe("dark");
    expect(localStorage.getItem("superbase-theme")).toBe("dark");
    expect(screen.getByRole("switch")).toBeChecked();
  });

  it("initializes from a stored dark preference", () => {
    localStorage.setItem("superbase-theme", "dark");
    render(<ThemeToggle />);
    expect(screen.getByRole("switch")).toBeChecked();
    expect(document.documentElement.getAttribute("data-theme")).toBe("dark");
  });
});

```tsx
"use client";
import { useEffect, useState } from "react";
import { Switch } from "@superbase/react";

const STORAGE_KEY = "superbase-theme";

function applyTheme(isDark: boolean) {
  document.documentElement.setAttribute("data-theme", isDark ? "dark" : "light");
}

export function ThemeToggle() {
  const [dark, setDark] = useState(false);

  useEffect(() => {
    const initial = localStorage.getItem(STORAGE_KEY) === "dark";
    setDark(initial);
    applyTheme(initial);
  }, []);

  function handleChange(next: boolean) {
    setDark(next);
    applyTheme(next);
    localStorage.setItem(STORAGE_KEY, next ? "dark" : "light");
  }

  return (
    <Switch checked={dark} onChange={handleChange} aria-label="다크 모드 토글" />
  );
}
```

          {radii.map((r) => (
            <div key={r.cssVar} style={{ display: "flex", flexDirection: "column", gap: "var(--spacing-1)", alignItems: "center" }}>
              <div
                style={{
                  width: 64,
                  height: 64,
                  background: "var(--color-background-subtle)",
                  border: "1px solid var(--color-border-default)",
                  borderRadius: `var(${r.cssVar})`,
                }}
              />
              <Text variant="caption" color="secondary">
                {r.name}
              </Text>
            </div>
          ))}
        </div>
      </section>
    </Stack>
  );
}
