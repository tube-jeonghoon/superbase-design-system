# Plan 10 — `@superbase/react-native` Icon (react-native-svg) 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `@superbase/icons`의 동일 패스 데이터를 소비하는 모바일 `<Icon name>` 컴포넌트를 `@superbase/react-native`에 `react-native-svg`로 추가한다(웹 Icon과 동일 API).

**Architecture:** RN Icon은 `react-native-svg`의 `<Svg><Path>`로 `iconPaths[name]`을 렌더한다(stroke 라인 스타일). `color` 기본값은 RN에 `currentColor`가 없으므로 토큰색(`ColorTextPrimary`). `react-native-svg`는 peerDependency(앱이 제공). 헤드리스 테스트는 `react-native-svg`를 DOM svg/path로 `vi.mock`해서 path의 `d`/stroke를 검증한다.

**Tech Stack:** React Native(peer) + react-native-web(test alias), react-native-svg(peer), TypeScript 5(tsc), Vitest + jsdom, Changesets. `@superbase/icons` + `@superbase/tokens/native` 소비.

> 전제: Plan 9가 `main`에 머지됨 — `@superbase/icons`가 `iconPaths`/`IconName`/`ICON_VIEWBOX`를 export. `@superbase/react-native`는 Text/Button/.../Spinner를 export(tsc 빌드, peer react-native, vitest가 react-native→react-native-web alias). `@superbase/tokens/native`는 `ColorTextPrimary` 등 hex 문자열 export. Node 22, pnpm 10.27.0.

- [ ] **Step 3: 실패 테스트 — `packages/react-native/src/Icon/Icon.test.tsx`**

```tsx
import { render } from "@testing-library/react";
import { vi } from "vitest";
import { iconPaths } from "@superbase/icons";

vi.mock("react-native-svg", () => ({
  Svg: ({ children, ...rest }: { children?: unknown } & Record<string, unknown>) => (
    <svg {...(rest as Record<string, unknown>)}>{children as never}</svg>
  ),
  Path: (props: Record<string, unknown>) => <path {...(props as Record<string, unknown>)} />,
}));

const { Icon } = await import("./Icon");

describe("Icon (RN)", () => {
  it("renders a path with the named icon's d", () => {
    const { container } = render(<Icon name="check" />);
    expect(container.querySelector("path")?.getAttribute("d")).toBe(iconPaths.check);
  });

  it("applies size to the svg and color to the path stroke", () => {
    const { container } = render(<Icon name="check" size={32} color="red" />);
    expect(container.querySelector("svg")?.getAttribute("width")).toBe("32");
    expect(container.querySelector("path")?.getAttribute("stroke")).toBe("red");
  });
});
```

> 주: `react-native-svg`를 DOM svg/path로 mock해 헤드리스에서 검증한다(react-native-svg의 web 빌드 의존 회피). `await import("./Icon")`은 mock이 적용된 뒤 Icon을 로드하기 위함이다 — vitest는 top-level await + hoisted vi.mock을 지원한다. (만약 동적 import가 문제되면 일반 `import { Icon } from "./Icon"`로 바꾸고 vi.mock이 hoist되는지 확인하라.)

- [ ] **Step 5: `packages/react-native/src/Icon/Icon.tsx`**

```tsx
import { Svg, Path } from "react-native-svg";
import { iconPaths, ICON_VIEWBOX, type IconName } from "@superbase/icons";
import { ColorTextPrimary } from "@superbase/tokens/native";

export interface IconProps {
  name: IconName;
  size?: number;
  color?: string;
  label?: string;
}

export function Icon({ name, size = 20, color = ColorTextPrimary, label }: IconProps) {
  const a11y = label
    ? { accessibilityRole: "image" as const, accessibilityLabel: label }
    : { accessibilityElementsHidden: true, importantForAccessibility: "no-hide-descendants" as const };
  return (
    <Svg width={size} height={size} viewBox={ICON_VIEWBOX} {...a11y}>
      <Path
        d={iconPaths[name]}
        fill="none"
        stroke={color}
        strokeWidth={2}
        strokeLinecap="round"
        strokeLinejoin="round"
      />
    </Svg>
  );
}
```

- [ ] **Step 6: 통과 확인** — Run: `pnpm --filter @superbase/react-native test` → PASS.

> 검증/적응: `accessibilityElementsHidden`/`importantForAccessibility`가 `SvgProps` 타입과 안 맞아 typecheck가 실패하면, 해당 a11y prop을 react-native-svg가 받는 형태로 조정(예: `accessibilityRole`만 사용하거나 prop 캐스트)하되 "label 있으면 접근명 노출, 없으면 장식" 의미는 유지하고 보고하라.
